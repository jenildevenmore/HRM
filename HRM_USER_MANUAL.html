Human Resource Management (HRM)

This manual explains how admins and users operate the HRM platform delivered as your branded product.

1. Introduction

The HRM application is a complete platform for employee lifecycle operations: onboarding, leaves, attendance, payroll, document handling, and activity logs.

AreaWhat it helps with
AdminClient setup, permissions, modules, branding, policy configuration.
HR / ManagerEmployee records, leave reviews, documents, payroll setup.
EmployeeLeave requests, attendance actions, personal document interactions.

2. Login

  1. Open /login/.
  2. Select login mode and client (if required).
  3. Enter username and password.
  4. On success, user is redirected to dashboard.
If login fails after deployment, check client activation key and .env values first.

3. Dashboard

The dashboard shows quick access cards and operational summaries. Use it as the daily entry point for module navigation.

  • Employees summary
  • Leave and attendance indicators
  • Quick links to major modules

4. Client Management

Client Management is used by superadmin to onboard organizations and configure tenant-level settings.

4.1 Creating Client

  1. Go to Clients from sidebar.
  2. Click Create Client.
  3. Fill name, domain, password, add-ons, role limit, and admin credentials.
  4. Save to provision the client and client admin.

4.2 Client Features

  • Add-on enable/disable per client
  • Role limit for custom client roles
  • Client-level app settings and branding controls

5. Employee Management

The Employee Management module helps you create, organize, and control your workforce records from one place.

5.1 Employee Administration

This section handles employee profile creation, role mapping, reporting structure, and record updates.

5.1.1 Creating Employee

To add a new employee, follow these steps:

  1. Access Employee Form
  • Navigate to Employees from the main sidebar.
  • Click Create Employee (or Add Employee) at the top area.
  1. Fill Employee Information

Enter the required employee details:

  • First Name: Employee first name (required)
  • Last Name: Employee last name (required)
  • Email: Unique employee email (required, used for mapping/login workflows)
  • Role: Select role (Employee / HR / Manager or configured role)
  • HR: Assign HR person (optional, based on organization flow)
  • Manager: Assign reporting manager (optional)
  • Joining Date: Select joining date (required)
  1. Save Record
  • Click Save or Create.
  • System creates employee profile and links organizational context for module access.

5.1.2 Employee Features

  • Role Assignment: Define access level through role and permission model
  • HR/Manager Mapping: Build approval and reporting hierarchy
  • Custom Fields Support: Capture additional client-specific employee data
  • Search & Filter: Quickly locate employees by name, email, or role
  • Profile Updates: Edit employee information without data loss
  • Safe Deletion Control: Remove employee records with permission-based restrictions

5.1.3 Editing Employee

  1. Go to Employees list.
  2. Select the employee and click Edit.
  3. Update fields such as role, manager, HR, or contact details.
  4. Click Update to save changes.

5.1.4 Viewing Employee Details

  1. Open Employees.
  2. Click the employee name or View action.
  3. Review profile summary, assignments, and related data.

5.1.5 Deleting Employee

  1. From employee list, click Delete on target record.
  2. Confirm action in prompt/modal.
  3. System removes the employee if permission and business rules allow.
In demo mode, create/update/delete buttons are clickable but write actions are blocked by design.

6. Roles & Permissions

Use this module to define role-based access for users and control feature visibility.

6.1 Creating Role

  1. Open Roles module.
  2. Click Create Role.
  3. Enter role name and permissions.
  4. Set add-ons if applicable and save.

6.2 Permission Group Assignment

  1. Go to Permissions.
  2. Create or edit permission group.
  3. Assign group to user profiles.

6.3 Permission System

The Permission System provides granular access control and security management across all HRM modules. It ensures users can only access features relevant to their role.

6.3.1 Understanding Permissions

HRM uses module-level permission keys with action scopes like:

  • View: Allows users to read module records and pages
  • Create: Allows users to add new records
  • Edit: Allows users to update existing records
  • Delete: Allows users to remove records
  • Approve / Manage: Allows workflow review or module-level administration where applicable

6.3.2 Permission Categories

Permissions are grouped by module so access can be configured precisely:

  • Employees: Employee profile create/edit/view/delete permissions
  • Leaves: Leave request lifecycle and approval permissions
  • Attendance: Attendance view/create/edit/delete controls
  • Holidays: Holiday setup and maintenance permissions
  • Shifts: Shift schedule management permissions
  • Bank: Employee bank details management permissions
  • Policy: Company policy create/edit/view/delete permissions
  • Documents: Document and upload-request permissions
  • Import / Export: Data import/export operation permissions
  • Custom Fields: Metadata field management permissions
  • Dynamic Models: Dynamic model/field/record permissions
  • Activity Logs: Audit log visibility permissions

By assigning permissions carefully, you can keep security tight while giving each user exactly what they need to do their work.

20. Activation Key (One-Time)

This product uses one-time client activation through CLIENT_EXECUTION_SECRET_KEY in .env.

  1. Seller provides one activation key for client purchase.
  2. Client adds key in .env.
  3. First valid run/login consumes key and activates client.
  4. Same key cannot activate another client.
# .env
CLIENT_EXECUTION_SECRET_KEY=replace-with-your-activation-key

21. Deployment (Linux VPS)

For production deployments, use Ubuntu + Gunicorn + Nginx with HTTPS.

Core commands

sudo apt update
sudo apt install -y python3 python3-venv python3-pip nginx
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python manage.py migrate
python manage.py collectstatic --noinput

Then configure systemd for Gunicorn and Nginx reverse proxy to 127.0.0.1:8000.

22. FAQ

Q: App opens but write actions do nothing.

A: Demo mode may be enabled. Set DEMO_MODE=False for normal production behavior.

Q: Login fails with activation key message.

A: Check CLIENT_EXECUTION_SECRET_KEY and restart app service.

Q: Static files missing in production.

A: Verify Nginx static alias and run python manage.py collectstatic --noinput.
