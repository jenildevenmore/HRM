HRM Project Installation & Setup

This page is a browser-ready version of the setup documentation for your Django-based HRM platform. It covers local installation, PostgreSQL configuration, environment variables, first-time setup, and production deployment preparation.

Project Type Unified Django application with UI and REST API
Database PostgreSQL is the active configured engine
Primary Run URL http://127.0.0.1:8000/login/

1. Introduction

The HRM Project is a Django-based Human Resource Management platform that combines a browser-based web interface with REST APIs for internal workflows and external integrations.

AreaDescription
FrontendServer-rendered pages for HR, managers, admins, and employees.
BackendDjango REST Framework APIs for authentication, module data, and integrations.
ModulesEmployees, leave, payroll, documents, attendance, settings, and activity logs.

2. Technology Stack

  • Python 3.10+
  • Django 5.2.12
  • Django REST Framework 3.16.1
  • PostgreSQL
  • Jinja2 templates
  • JWT authentication
  • ReportLab for PDF generation

3. System Requirements

Minimum Recommended Environment

  • Windows 10/11, Ubuntu 22.04+, or any modern Linux server
  • Python 3.10 or above
  • pip package manager
  • PostgreSQL 13+ recommended
  • 4 GB RAM minimum
  • 2 GB free disk space minimum

Production Recommendations

  • Nginx or Apache as reverse proxy
  • Gunicorn or another WSGI server
  • HTTPS-enabled domain
  • SMTP account for email delivery

6. Installation Guide

6.1 Clone or Copy the Project

cd C:\
git clone <your-repository-url> hrm_project
cd C:\hrm_project

If the project is shared as a ZIP file, extract it into C:\hrm_project.

6.2 Create a Virtual Environment

cd C:\hrm_project
python -m venv .venv

Activate it on Windows:

.\.venv\Scripts\activate

Activate it on Linux:

source .venv/bin/activate

6.3 Install Dependencies

pip install -r requirements.txt
The current requirements.txt does not include a PostgreSQL adapter package. Install psycopg[binary] or psycopg2-binary before running migrations if your environment does not already provide it.

7. Environment Configuration

Create or update the root environment file at c:\hrm_project\.env.

DJANGO_SECRET_KEY=change-this-to-a-secure-value
DJANGO_DEBUG=True
DEMO_MODE=False
DJANGO_ALLOWED_HOSTS=127.0.0.1,localhost
DJANGO_CSRF_TRUSTED_ORIGINS=http://127.0.0.1:8000,http://localhost:8000
DJANGO_TIME_ZONE=Asia/Kolkata

DJANGO_PG_NAME=HRM
DJANGO_PG_USER=postgres
DJANGO_PG_PASSWORD=postgres
DJANGO_PG_HOST=localhost
DJANGO_PG_PORT=5432

DJANGO_STATIC_URL=/static/
DJANGO_STATIC_ROOT=
DJANGO_MEDIA_URL=/media/
DJANGO_MEDIA_ROOT=

EMAIL_BACKEND=django.core.mail.backends.smtp.EmailBackend
EMAIL_HOST=smtp.gmail.com
EMAIL_PORT=587
EMAIL_USE_TLS=True
EMAIL_HOST_USER=
EMAIL_HOST_PASSWORD=
DEFAULT_FROM_EMAIL=

BACKEND_API_URL=http://127.0.0.1:8000
USE_INTERNAL_API=True
FRONTEND_BASE_URLS=http://127.0.0.1:8000
FRONTEND_BASE_URL=http://127.0.0.1:8000
CLIENT_EXECUTION_SECRET_KEY=

SESSION_COOKIE_SECURE=False
CSRF_COOKIE_SECURE=False
SECURE_SSL_REDIRECT=False
PostgreSQL is the active configured database engine in the current codebase. If DJANGO_PG_PORT is not set, the application defaults to port 5433.

8. Database Setup

8.1 Create PostgreSQL Database

CREATE DATABASE HRM;
CREATE USER hrm_user WITH PASSWORD 'strong-password';
GRANT ALL PRIVILEGES ON DATABASE HRM TO hrm_user;

Then update your .env values with the real database credentials.

8.2 Run Migrations

cd C:\hrm_project\hrm_project
..\.venv\Scripts\python.exe manage.py migrate

If your shell already has the environment activated, you can simply run:

cd C:\hrm_project\hrm_project
python manage.py migrate

8.3 Optional: Create Admin User

python manage.py createsuperuser

9. Running the Application

cd C:\hrm_project\hrm_project
python manage.py runserver 0.0.0.0:8000

After startup, open the following routes in a browser:

  • http://127.0.0.1:8000/login/
  • http://127.0.0.1:8000/api/
  • http://127.0.0.1:8000/admin/

10. First-Time Application Setup

  1. Create a superuser using createsuperuser.
  2. Log in through Django admin or the HRM UI.
  3. Create a client or company record.
  4. Create user roles and permission groups.
  5. Add employees and assign roles.
  6. Configure leave types, holidays, shifts, banks, payroll, and documents.

The UI also includes an onboarding route:

/onboarding/setup-org/

11. Create Client Admin

The project includes a helper command to create a client-specific admin user from the command line.

python manage.py create_client_admin --username admin1 --password StrongPass123 --email admin@example.com --client-id 1
  • Creates a Django user
  • Links that user to a client
  • Assigns the role admin

13. Production Deployment Checklist

  • Set DJANGO_DEBUG=False
  • Use a strong DJANGO_SECRET_KEY
  • Set real domains in DJANGO_ALLOWED_HOSTS
  • Set HTTPS domains in DJANGO_CSRF_TRUSTED_ORIGINS
  • Set SESSION_COOKIE_SECURE=True
  • Set CSRF_COOKIE_SECURE=True
  • Set SECURE_SSL_REDIRECT=True
  • Configure valid SMTP credentials
  • Run static file collection
  • Serve media files correctly
  • Place Gunicorn behind Nginx or Apache
python manage.py collectstatic --noinput

15. Troubleshooting

15.1 Database Connection Error

  • Verify the PostgreSQL service is running.
  • Recheck database name, username, password, host, and port.
  • Confirm the values in .env match the real server.

15.2 Static Files Not Loading

  • Check DJANGO_STATIC_URL and DJANGO_STATIC_ROOT.
  • Verify reverse proxy static mapping in production.
  • Run collectstatic.

15.3 Media Files Not Opening

  • Check DJANGO_MEDIA_URL and DJANGO_MEDIA_ROOT.
  • Verify path permissions on the server.

15.4 Email Not Sending

  • Verify SMTP host, port, TLS, username, and password.
  • Use an app password when required by the provider.

15.5 Login or Permission Issue

  • Check that the user exists.
  • Check profile and client mapping.
  • Verify role and permission-group assignment.
